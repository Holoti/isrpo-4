## 1. Цели и задачи тестирования
Определение основных целей тестирования и задач, которые необходимо достичь в процессе тестирования.

- Проверить правильность реализации функций вычисления площади и периметра в базовых геометрических модулях: `circle.py`, `rectangle.py`, `square.py`, `triangle.py`.
- Обнаружить и документировать возможные ошибки на краевых значениях (0, дробные, очень большие числа).
- Обеспечить базовую автоматизацию проверки кода с помощью unit-тестов (модульного тестирования) и подготовить отчёт о проделанной работе.

## 2. Описание тестируемого продукта
Обзор функциональности, особенностей и требований к продукту, которые должны быть протестированы.

- Проект реализует небольшую библиотеку утилитарных функций для вычисления площади и периметра простых геометрических фигур.
- Ключевые файлы/функции:
	- `circle.py`: area(r), perimeter(r)
	- `rectangle.py`: area(h, w), perimeter(h, w)
	- `square.py`: area(a), perimeter(a)
	- `triangle.py`: area(a, h), perimeter(a, b, c)

## 3. Область тестирования
Определение конкретных функций, модулей или компонентов продукта, которые будут исследованы в тестирование.

- Проверка корректности вычислений для каждой функции area(...) и perimeter(...) в модулях выше.
- Проверка краевых значений: нулевые входы, дробные значения (float), и контрольные величины (например, радиус 1 → площадь π).

## 4. Стратегия тестирования
Описание общего подхода к тестированию, включая методы, техники и типы тестирования, которые будут использоваться, например, функциональное тестирование, тестирование производительности, тестирование безопасности и т. д.

- Подход: модульное тестирование (unit tests) с `unittest` — быстрые, детерминированные, воспроизводимые тесты.
- Каждый файл модуля имеет файл тестов с набором независимых тест-кейсов (`test*.py`).
- Фокус на функциональном тестировании и проверке граничных случаев; нагрузочное/производительное тестирование в данном случае не применимо к простым числовым функциям.

## 5. Критерии приемки
Определение условий и критериев, которые должны быть выполнены для успешного завершения тестирования и приемки продукта.

- Все написанные unit-тесты должны проходить при запуске.
- Тесты должны покрывать все реализованные простейшие функции area() и perimeter() во всех модулях.
- Тесты должны быть детерминированы и не зависеть от внешнего состояния или порядка выполнения.

## 6. Ожидаемые результаты
Указание ожидаемых результатов тестирования, таких как отчеты о дефектах, статусы тестирования, метрики качества и другие соответствующие данные.

- Детализированный отчёт о запуске тестов: сколько тестов запущено, сколько прошло/упало.
- В случае провала — список упавших тестов, ожидаемое vs реальное значение и ссылка на файл/строку.

## 7. Актуальные результаты тестирования

- Написаны и добавлены следующие файлы с тестами:
	- `circle_test.py` — проверка area(r) и perimeter(r). Тесты включают r=0, r=1 и дополнительные контрольные значения, связанные с π.
	- `rectangle_test.py` — проверка area(h, w) и perimeter(h, w) для нулевых и положительных значений, а также тест для квадрата (h == w).
	- `square_test.py` — проверка area(a) и perimeter(a) для n=0, n=1, дробных значений и больших чисел.
	- `triangle_test.py` — проверка area(a, h) (включая нулевые параметры) и perimeter(a, b, c), включая известные наборы значений (например, 3-4-5).

- Статус запуска тестов: запущено автоматически с помощью команды

```bash
python -m unittest discover -v
```

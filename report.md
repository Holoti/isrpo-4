# Отчет

Ссылка на репозиторий:
https://github.com/Holoti/isrpo-4

## 1. Цели и задачи тестирования
Определение основных целей тестирования и задач, которые необходимо достичь в процессе тестирования.

- Проверить правильность реализации функций вычисления площади и периметра базовых геометрических фигур
- Проверить поведение на corner-кейсах (нулевые, отрицательные значения)
- Автоматизировать тестирование с помощью unittest

## 2. Описание тестируемого продукта
Обзор функциональности, особенностей и требований к продукту, которые должны быть протестированы.

- Проект реализует функции вычисления площади и периметра базовых геометрических фигур
- Функции должны возвращать верные значения для нормальных случаев и обрабатывать corner-кейсы

## 3. Область тестирования
Определение конкретных функций, модулей или компонентов продукта, которые будут исследованы в тестирование.

- `circle.py`: area(r), perimeter(r)
- `rectangle.py`: area(h, w), perimeter(h, w)
- `square.py`: area(a), perimeter(a)
- `triangle.py`: area(a, h), perimeter(a, b, c)

## 4. Стратегия тестирования
Описание общего подхода к тестированию, включая методы, техники и типы тестирования, которые будут использоваться, например, функциональное тестирование, тестирование производительности, тестирование безопасности и т. д.

Тестирование производительности и тестирование безопасности не имеют смысла в данном проекте ввиду его простоты.

Тестирование функциональности заключается в написании и запуске unit-тестов для каждой функции проекта.

## 5. Критерии приемки
Определение условий и критериев, которые должны быть выполнены для успешного завершения тестирования и приемки продукта.

- Правильный ответ для всех нормальных случаев
- Обработка corner-кейсов

## 6. Ожидаемые результаты
Указание ожидаемых результатов тестирования, таких как отчеты о дефектах, статусы тестирования, метрики качества и другие соответствующие данные.

- Отчёт о запуске тестов: количество запущенных, количество успешно пройденных
- Информация о неудачных тестах: их количество, описание, фактические значения

## 7. Актуальные результаты тестирования

- Написаны следующие файлы с тестами:
	- `test_circle.py`
	- `test_rectangle.py`
	- `test_square.py`
	- `test_triangle.py`

Тесты запускаются автоматически с помощью команды:
```bash
python -m unittest discover -v
```

Результат запуска тестов:
	- Запущено: 34 теста
	- Успешно: 25 тестов
	- Проавлено: 9 тестов

Провалены тесты, проверяющие негативные значения. Программа не обрататывает такие значения отдельно и выводит числовые значения вместо ожидаемого None.
